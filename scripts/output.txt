### 3.1 Safety architecture

In Figure 1 the safety architecture is shown.

Input
Processing
Output
![img-0.jpeg](img-0.jpeg)

Figure 1: Safety architecture

The definitions used in Figure 1 are shown in Table 4.
Table 4: Definition safety architecture

| Definition | Description |
| :--: | :-- |
| Level 1 | Level 1 contains the overall functional software of this project. It also contains all diagnosis and <br> limp home modes identified in System-FMEA. The software and the components of Level 1 are <br> developed according to ASIL QM. Therefore no functional software, no diagnosis and limp <br> home mode of Level 1 can be used for the safety concept to prevent a safety goal violation. |
| Level 2 | Level 2 contains the safety function which are preventing a safety goal violation. In level 2 also <br> the input signals and the output signals used for the safety functions are calculated. |
| Level 3 | Level 3 is the monitoring of the processing unit and is split in two parts. <br> One part is on the monitoring unit which is monitoring the processing unit by generating <br> question to the processing unit and checking for correct answers within a specified time <br> window. <br> The other part is on the micro controller and checks the micro controller related diagnosis like <br> RAM, ROM, etc |

# 4.4 Safe state 

### 4.4.1 Entry condition

The safe state "active open clutch" is called by level 2.

### 4.4.2 Actions

If the safe state "active open clutch" is triggered the following actions are carried out:

- The CAN communication
- The bridge for the shift motor $1 / 2$ is disabled by the micro controller
- The clutch motor $1 / 2$ is activated in opening direction for a certain time period by the micro controller


### 4.4.3 Safe state â€žOutput state tristate (OST)"

### 4.4.4 Entry condition

The safe state "Output state tristate (OST)" is called by level2/level 2s/3.

### 4.4.5 Actions

If the safe state "Output state tristate (OST)" is triggered the following actions are carried out:

- The bridge for the shift motor $1 / 2$ is disabled by the micro controller or the monitoring unit or the supply monitoring $\mu \mathrm{C}$ (depends on the unit which detects the error)
- The bridge for the clutch motor $1 / 2$ is disabled by the micro controller or the monitoring unit or the supply monitoring $\mu \mathrm{C}$ (depends on the unit which detects the error)


### 4.4.6 Safe state - called by level 2

If vi aoc possible is equal level 2 carries out the following actions:

- If the allowed fault detection time (for active opening clutch) is reached level 2 calls safe state "Active open clutch (AOC)".

If vi aoc possible is equal to False level 2 carries out the following actions:

- level 2 reduces the fault detection time so that the safe state is reached within the process safety time if only passive opening of the clutch is possible
- If the allowed fault detection (for passive opening clutch) calls safe state "Output state tristate (OST)".
