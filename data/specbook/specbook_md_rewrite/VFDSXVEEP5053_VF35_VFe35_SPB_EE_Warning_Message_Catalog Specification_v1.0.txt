3.2 Signal Mechanism.  

There are three mechanisms to display a telltale/indicator/warning message on Headunit (MHU):  
1. **By Keep Alive Monitoring:**  
   If the sender stops sending alive messages after several cycles, the MHU detects fault and displays a telltale/indicator/warning message according to related functions of the sender. If the sender follows VF warning message structure (as defined by section 3.3), a dedicated Keep Alive CAN message is not needed.  
   
2. **By VF Warning Message:**  
   The sender sends MHU a warning message based on the VF structure defined in section 3.3.  

3. **By Carry-Over or Indicator Message:**  
   - **Status CAN Message:** The sender sends a Status CAN message to indicate a state of its operation (e.g., turn indicator, state light system, door ajar state, ADAS state).  
   - **Carry-Over Message:** If the ECU is a carry-over part unable to apply VF warning message structure, the MHU follows the ECU’s existing mechanism.  

---

3.3 VINFAST Warning Message Definition.  

VINFAST warning messages follow the E2E profile structure defined by VF CAN specification.  

**Vinfast Warning Message Name Convention:** `<ECU name>_WarnMsg`.  
Error signal names (e.g., Wflag1, Wflag2) must be renamed to meaningful names by SOP milestone.  

**Value Convention for Single Warning Flag (1 Bit):**  
- `0x0`: Normal.  
- `0x1`: Event Trigger.  

Each warning message bit flag maps only one warning.  

An ECU warning ID can encode up to 52 warning messages (from bit 4 to bit 63). If there are more than 52 messages, additional IDs must be defined (e.g., BCM_WarnMsg1, BCM_WarnMsg2).  

**Special Cases:**  
Adjacent bits may encode groups or levels of warnings (e.g., 12th and 13th bits encode a warning set with four warnings/levels).  

**Unused Bits:**  
Unused bits allow reserving up to 2 bytes for future changes.  

**Transmit Type:**  
Transmit type is **cyclic with repetition on event**:  
- After an event is triggered (`SET`), send two repetitions (`nRep = 2`) at intervals (`tRep = 100ms`).  
- Messages are sent cyclically (`tCyclic = 1000ms`).  

If an event is unset, corresponding flags reset in the next cycle.  

Because the MHU uses VF warning messages to monitor ECU alive status, separate alive messages (as per section 3.2) are unnecessary.

3.4 Warning Message Strategy.

A warning message signal shall bind the following information:
- Warning message content.
- Telltale/Indicator.
- Chime sound.

3.4.1 Warning Message Category.

Warning messages displayed in the driving information region are grouped into three different categories:
1. **Category 0 (P0):** Alerts for system status that require immediate attention or a change in behavior. It suppresses all other lower priority messages. Alert messages are contextual and aren't stored.
2. **Category 1 (P1):** High-priority warning messages that impact vehicle safety or human life. These messages are displayed continuously and remain visible until the malfunction has been rectified. If multiple malfunctions occur simultaneously, the messages are displayed consecutively. These messages can be hidden for approximately 8 seconds, after which they are displayed again automatically. These messages shall also be stored and can be viewed in the warning messages list pop-up.
3. **Category 2 (P2):** Low-priority messages indicating less serious issues. These messages shall be displayed once and hidden automatically after 16 seconds. They will also be stored and can be seen in the warning messages list pop-up.
4. **Category 3 (P3):** Service reminder messages related to the start or end of the trip.

3.4.2 Warning Message Priority.

Warning message priorities shall be ranked using the following rules:
- Category priority order: P3 < P2 < P1 < P0 (ascending priority).
- Each message is assigned a unique priority number, with lower numbers indicating higher priority.

3.4.3 Telltales.

Telltales function as indicators for warning messages and serve two types:
- **Dedicated Telltales:** Mandatory indicators required by regulations and safety requirements.
- **Dynamic Telltales:** Additional indicators displayed dynamically alongside warning messages, as described in Figure 5.

If a warning message only indicates a status (such as Turn Left, Turn Right, High Beam, Front Fog, or Rear Fog), it does not include message content. Conversely, for system failures or malfunctions, a warning message will consist of both a telltale and message content to explain the warning to the driver.

3.4 Warning Message Strategy.

A warning message signal shall bind the following information:
- Warning message content.
- Telltale/Indicator.
- Chime sound.

3.4.1 Warning Message Category.

Warning messages displayed in the driving information region are grouped into three different categories:
- **Category 0 (P0)**: Alerts for system status that require immediate attention or a change in behavior. These alerts suppress all other lower-priority messages. These are contextual alert messages and are not stored.
- **Category 1 (P1)**: High-priority warning messages that impact vehicle safety or human life. These messages are displayed continuously and remain visible until the malfunction has been rectified. If several malfunctions occur simultaneously, they are displayed consecutively. The messages can be hidden for approximately eight seconds but will reappear automatically. These messages are stored and can be accessed via the warning messages list pop-up.
- **Category 2 (P2)**: Low-priority, less serious warning messages displayed once and hidden automatically after 16 seconds. These messages are stored and can be accessed via the warning messages list pop-up.
- **Category 3 (P3)**: Service reminder warning messages related to trip initialization or completion.

3.4.2 Warning Message Priority.

The priority of a message shall be ranked as follows:
- **Category:** Priority rank in ascending order: P3 < P2 < P1 < P0.
- **Unique Priority Number Assignment:** Each message is assigned a unique priority number. Lower numbers correspond to higher priority.

3.4.3 Telltales.

- If a warning message only indicates a status, it does not include message content. For example: Turn left, Turn right, High Beam, Front Fog, Rear Fog.
- If a warning message indicates system failure or malfunction, both telltale and content are included to inform the driver of the warning's meaning.

MHU shall feature two types of telltales:
- **Dedicated Telltales:** Mandatory telltales required by regulatory and safety requirements.
- **Dynamic Telltales:** Additional telltales dynamically displayed alongside warning messages as shown in Figure 5.

3.4.4 Chime Sound.

A warning signal shall include an audible chime when required. Types of chime sound include:
- **NG:** No gong - no acoustic output.
- **Tick/Tock:** Tick/Tock sound for Turning Indicator/Hazard Indicator.
- **PDC:** Parking Distance Control acoustical warnings categorized as follows: PDC_SelfTest < PDC_1Hz < PDC_2Hz < PDC_4Hz < PDC_Continuous.
- **WMG:** Warning message gong - single acoustical warning output.
- **IMG:** Intermittent message gong - multiple acoustical warnings output.
- **LWG:** Left warning message gong - multiple beeps.
- **RWG:** Right warning message gong - multiple beeps.

**Sound Priority Order:** Tick/Tock < WMG < IMG < PDC_SelfTest < RWG < LWG < PDC_1Hz < PDC_2Hz < PDC_4Hz < PDC_Continuous.

3.4.5 Displaying Warning Message Strategy.

MHU shall display Telltales/Indicators/Warning messages according to the latest HMI specification.

3.5 Description of Warning Catalog.

**Sender:** Identifies the sender Control Unit name.
**Warning Message ID:** Each message shall be assigned a unique warning message ID.
**Category:** Defines warning categories as described in section 3.4.1:
- Category 1: P0 - Alert.
- Category 2: P1 - High priority.
- Category 3: P2 - Low priority.
- Category 4: P3 - Service reminder warning message.
**Priority:** Assigns a unique number where lower numbers indicate higher priority. Default value is 0.
**Cause:** Identifies the root cause of the warning message (e.g., driver door open and light not switched off).

## Repetition at the End of Trip
Existing messages shall be repeated at the end of the trip (after Ignition OFF) if defined. Options include:
- **No**: No repeat.
- **Yes**: Repeat at the end of the trip.

### Service Levels
Defines the car service support level for the warning:
- **S0**: Not service relevant.
- **S1**: Self-fix.
- **S2**: Book a service appointment.
- **S3**: Call roadside assist.

### Requirements for Head Unit (MHU)
The Head Unit (MHU) shall:
1. Display the received signal conditions leading to a warning.
2. Define special behaviors or logic for displaying on the HMI, if required.

### Symbol Telltale/Color
Shows the graphic and color of the telltale.

### Durable/Blinking
Specifies if the telltale is blinking (frequency) or highlighted as durable.

### Special Display
Defines outstanding UI elements that are not part of the telltale area or car buddy area. Examples include:
- Passenger airbag OFF.
- Battery level.
- Cold weather indication.

### Car Buddy Indicator
Illustrates car status or driving status in the Car Buddy Indicator region (e.g., ADAS information) visually, if active.

### Warning Message Symbol/Color
Displays the graphic and color of the symbol with the warning message, if active.

## Warning Message Formats
Two formats are available for warning messages:
1. **SHORT**: Maximum 54 characters including whitespace characters.
   - Displayed in the warning message area.
2. **LONG**: Maximum 160 characters including whitespace characters.
   - Displayed when the driver opens the detailed warning.

### Convention
Format: `<SHORT> | <LONG>`

### Language
Languages supported:
1. **English** (Mandatory).
2. Vietnamese and other languages translated by EE Vinfast.

## Acoustic Warning Signals
Defines the chime sound or the condition/logic for playing chime sounds. 
- If the chime is multiple gongs, `tchime_timeout` shall be defined.
- If no timeout is defined, the gong sound shall continue until the warning disappears.

### Types of Chime Sounds
- **NG**: No gong (no acoustic output).
- **Tick/Tock**: Used for Turn Indicator/Hazard indicators.
- **PDC**: Parking Distance Control acoustical warning. Includes:
  - `PDC_SelfTest < PDC_1Hz < PDC_2Hz < PDC_4Hz < PDC_Continuous`.
- **WMG**: Warning message gong (single acoustical gong).
- **IMG**: Intermittent message gong (multiple gongs).
- **LWG**: Left warning message gong (multiple beeps).
- **RWG**: Right warning message gong (multiple beeps).

### Sound Priority
Order of priority:  
`Tick/Tock < WMG < IMG < PDC_SelfTest < RWG < LWG < PDC_1Hz < PDC_2Hz < PDC_4Hz < PDC_Continuous`.

## Timeframe
Defines the timeframe when a warning message is valid, depending on vehicle state, as per the latest E/E Architecture.

### Timeframe Convention
Format: `<B|E>.<From Clamp>_<B|E>.<To Clamp>`
Definitions:
- **B**: Begin of Clamp.
- **E**: End of Clamp.
Page 16
For example:
- **B.CL30s_B.CL15:** The message can be set beginning from clamp 30s on until clamp 15 on.
- **B.CL30s_E.CL15:** The message can be set beginning from clamp 30s on until clamp 15 off.
- **B.CL30s_E.CL30s:** The message can be set beginning from clamp 30s on until clamp 30s off.
- **B.CL15_E.CK15:** The warning message can only be set beginning from clamp 15 on until clamp 15 off.
- **B.CLR_E.CLR:** The warning message can only be set as long clamp R is on.

### Special timeframe:
- **CAN wake-up:** The warning can be set if MHU is wake up by CAN.
- **D_D:** The warning message can only be set during driving, if vehicle speed is > 5 km/h and CL15 is on.
- **CL.15:** The warning message can only be set beginning from clamp 15 on until clamp 15 off.
- **CL.R:** The warning message can only be set beginning from clamp R on until clamp R off.

### Remarks:
Optional remarks for each warning.

### Special behavior for Warning:
- A warning works only during its defined timeframe.
