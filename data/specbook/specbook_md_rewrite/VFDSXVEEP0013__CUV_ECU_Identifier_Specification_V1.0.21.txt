### 3. System description  
#### 3.1 Address Scheme  
The local node identifications of ECU, and consequently the global node identifications, must be allocated uniquely within the entire vehicle network. In accordance with the determinations, numeric values in the range from 0 to (2b - 1) are used for this purpose, where b is the number of bits used for node identification/addressing. Currently, it is assumed that a maximum of 256 ECUs (real or virtual) will be addressed in one and the same OEM-vehicle. Therefore, only one byte (b = 8 bits) shall be used to address an ECU.  

Group addresses are provided for special applications by the system responsible. It depends on the selected transformation for node identification into the message header, whether the local and global node identifications are equal.  

---

#### Node Identification / Address Range Interpretation  
- **0 = 0x00:** Reserved; don’t use to identify an ECU.  
- **1 … 127 = 0x01 … 0x7F except 51 = 0x33:**  
  It is assumed that there will be no more than 127 real ECUs in an OEM-vehicle and, therefore, numbers 1 up to 127 shall be used to identify/address real ECUs in the OEM-vehicle.  
  The identification/address number 51 = 0x33 may be used to address/identify the group of all OBD-relevant ECUs in the OEM-vehicle. Therefore, this identification/address number 51 = 0x33 shall not be used to identify/address another ECU.  
- **128 … 254 = 0x80 … 0xFE:**  
  In any case, some real ECUs embed virtual ECUs, off-board test equipment (e.g., diagnostic tester), or functional groups of network nodes (ECUs), which can be addressed/identified using numbers 128 up to 254.  
- **255 = 0xFF:**  
  Shall be used to address the group of all identifiable/addressable nodes in the OEM-vehicle.

### 3.1.7 Development (e.g. CCP/XCP Master/Slave) – Identifiers  
The CAN-message identifiers for ECU development purposes (XCP/CCP-master and slave) shall be calculated as:  
- **REQ_ID**: Request-message = “XCP-Master message-ID” = 0x7AF + ECU_Address  
- **RSP_ID**: Response-message = “XCP-slave message-ID” = 0x780 + ECU_Address  

Maximum 94 CAN identifiers, ranging from 0x780 to 0x7DE, can be used for development purposes. These identifiers cater to supplier-specific protocols or transport-protocols according to ISO 15765-2, requiring two CAN identifiers for request and response.  

In special cases, the Network Design Team Leader may assign the message ID for development purposes.

3.2.1. Diagnostic message on CAN.

Diagnostic/transport protocol-messages are not sent cyclically as they answer to spontaneous diagnostic requests. Every control unit has one CAN-Identifier, defined in table at 4, which is used by the control unit when sending answers to requests from Tester.
