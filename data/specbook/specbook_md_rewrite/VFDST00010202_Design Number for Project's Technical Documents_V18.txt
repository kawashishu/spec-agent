Scope and purpose.  
This standard covers the structure and the usage for the project’s technical document number at VinFast as mentioned in kind of document (item 2).  

Also, the document shall be uploaded and triggered on the Teamcenter system for products such as Battery pack, ESS, Escooter, Golfcart, and Cell product at VF.  

The documents shall be uploaded and triggered on the Teamcenter system at VinFast.  

Abbreviations:  
- **VF:** VinFast  
- **SOR:** Scope of Requirements  
- **SPB:** Spec Book (Specification book)  
- **DVPR:** Design Validation Plan & Report  
- **TGSS:** Tool Grain Specification Sheet  
- **TE:** Test report  
- **FMEA:** Failure Mode Effect Analysis  
- **CMFS:** Color, Material, Finishing Spec book  
- **DTSS:** Dimension Technical Specification Sheet  
- **VTSS:** Vehicle Technical Specification Sheet  
- **VFFS:** VinFast Functional Safety document  
- **EOL:** Spec End of Line  
- **DOE:** Design of Experiment  
- **MSDS:** Material Safety Data Sheet

3 Naming convention.

**Attribute Specifications:**

1. **ID number**
   - **Digit:** 1-13
   - **Length:** 13 characters
   - **Requirements & Example:** Its structure follows the General Layout (Example: VFSRAABIW0001).

2. **Version**
   - **Digit:** 15-16
   - **Length:** 2 characters
   - **Requirements:** Range from 01 to 99.

3. **Model name**
   - **Digit:** 18-25
   - **Length:** Maximum 8 characters.
   - **Requirements:** Consistent capital letters must be used for all model names (Example: VFe34 / VFe34RHD).

4. **Document type**
   - **Digit:** 27-30
   - **Length:** Maximum 4 characters.
   - **Requirements & Example:** Example types include SPB, DVPR, VFTE, FMEA, etc.

5. **File name**
   - **Digit:** 32-256
   - **Length:** Maximum 225 characters.
   - **Requirements:**
     - Spaces must be replaced by hyphens (-) and underscores (_).
     - No special characters such as #, %, ?, /, \, (, ), @, [, ].
     - English language strictly (Chinese characters must not be used).
     - Example format: `XXXXXXXXXXXXX_XX_XXXXXXXX_XXXX_XXXXXXXXXXXX.....................................XXXXXXXX.pdf/xlsx/doc`.

**Document Naming Components:**
- **Document Code:** Refers to the structured ID.
- **Model Name:** As defined above (Example: VFe34).
- **Version:** Utilizes a two-digit version indicator.
- **File Name:** Must include the module name applicable to the document.
- **Document Type:** Corresponds to predefined four-character document identifiers.

3.1 VFSR structure. 

(1) Description name in Teamcenter: VFSRAABIW0001/01-VFe34_SOR_BIW_ Brief description (Model code, SOR, Module name must be included in the name).  
(2) SOR file name: VFSRAABIW0001_01-VFe34_SOR_BIW_ Brief description (Model code, SOR, Module name must be included in the name).  
- VFSRAABIW0001: Body In White Scope of Requirements for VFe34 project with number 0001.  
- 01: version of SOR in Teamcenter.  
Note: VFSR template reference to VFDST00029903 in Teamcenter.

3.2 VFDS structure. 

Structure templates shall develop and build up from SOR contents after SOB and input additional specifications/materials, requirements/testing items. Details become SPB and upload with prefix VFDS*.  

(1) Description name in Teamcenter: “VFDSAACHS0001/01-VFe34_SPB_CHS_Brief description” (Model code, SPB, Module name must be included in the name).  
(2) Specbook file name: “VFDSAACHS0001_01-VFe34_SPB_CHS_Brief description” (Model code, SPB, Module name must be included in the name).  
- VFDSAACHS0001: Specification book component Chassis, VFe34 project with number 0001.  
- 01: version of spec book in Teamcenter.  

Ensure that the part number be copied and pasted to the folder “Spec Item” of SPB Number version respectively.  
Note:  
- SPB for each project must be uploaded and released on Teamcenter belonging to the module team. The Module Leader must define quantity and ensure the correctness of content and requirements in the spec book when uploaded in Teamcenter systems.  
- SPB shall be released in Teamcenter by ECR process.  
- SPB template reference to VFDST00000007 in Teamcenter.
